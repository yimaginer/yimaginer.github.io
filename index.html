银行现状：

金融业降薪，但总体上银行还是不错的选择。此外三桶油，国网，烟草也是不错的！

  • 总行：核心岗位最多，科技岗核心研发都是在总行或者总行直属的研发中心（软开中心、大数据中心等)，有些银行可能会有加班的情况存在，薪资较高但是年终奖占比大、成长机会较多。
  • 省分行：有一定数量的科技岗，大部分银行省分科技岗较为轻松稳定，薪资一般都低于总行，且成长较为受限。
  • 省分以下：省分以下的分行一般来说科技岗较少甚至没有，大部分都是柜员、销售等。薪资和发展较为有限，且一线业务岗可能会面临绩效压力。

第一梯队稳定性最高，薪资最低。第二梯队高一些，中信直属研发中心转正后一线可达 30W，成都可到 25W

科技岗门槛

总行：总行信息部+直属研发中心，科技岗位多，但是竞争激烈。四大银行/招商银行总行信息部一般学历最好本硕985/QS前50以上且有相关实习。直属研发中心其他银行总行对院校要求可能会稍低。
省分行：科技岗一般来说要求本硕有一个是211/双一流，本硕双非+多段优质实习也有机会。管培生岗位对院校要求会稍微高一些。
省分以下：一般本科以上，对院校没有的特别死，但是近几年稍微卷一些，硕士应聘人数在增加。

  3. C++ 多态

    • 静态多态：函数重载，运算符重载，泛型多态(类模板和函数模版)。特点：编译时
    • 动态多态：也称运行时多态，主要是通过虚函数实现，允许在子类中重写父类的方法，对函数调用的解析在运行时进行

  4. 动态多态的实现机制

    • 构建时：父类将需要具体实现的方法定义为虚函数，实现共同逻辑或不实现任何逻辑（纯虚函数）。子类根据具体行为对同名虚函数进行覆盖。

    • 编译时：为处理虚函数，编译器在编译时候在子类内存空间的首地址放入虚函数表指针。虚函数表指针指向子类的虚函数表，这个虚函数表继承自父类，然后在这个虚函数表中作两个事，覆盖父类的同名虚函数地址，在末尾增加自己新定义的虚函数。

    • 运行时：用父类指针指向子类对象，想当于把子类对象当做父类对象去用，此时按照访问父类虚函数的数据流去访问，由于虚函数已经被覆盖，自然访问到的就是子函数的虚函数了。可以验证的是如果用这种方式访问子类新定义的虚函数将会出错。

      所以本质上子函数覆盖了的虚函数其实相当于生成了不同名函数，只不过编译器通过虚表把执行流引到了对应的虚函数位置。这样程序员在编码的时候就可以减少工作量了。

  5. 抽象类

    • 含有至少一个纯虚函数的类，不能被直接实例化，通常用于定义一个接口或基类。
    • 纯虚函数是一种没有实现的虚函数，它的存在是为了强制派生类提供这些函数的具体实现。

  6. 泛型编程

    • 主要通过模版机制实现，允许编写和类型无关的代码，模版可以是函数模版或类模版

  7. 函数重载

    • 参数的个数，类型，或顺序不同
    • 返回值：参数完全相同，只有返回值类型不同不能构成重载。
    • 类型：构造函数重载，运算符重载，函数重载，成员函数重载
    • 运算符重载：+ - * / == != << >> () ++ –

  8. C++ 类中的四种成员

    • 非静态数据成员被放在每一个对象体内作为对象专有的数据成员。
    • 静态数据成员被提取出来放在程序的静态数据区内(和全局变量一样的位置)，为该类所有对象共享，只存在一份。
    • 静态和非静态成员函数最终都被提取出来放在程序代码段中并为该类所有对象共享，因此每个成员函数也只能存在一份代码实体。C++中类的成员函数都是保存在静态存储区中，静态成员函数也是保存在静态存储区中，他们都是在类中保存同一个备份。
    • 说明：构成对象本身的只有数据，任何成员函数都不隶属于任何一个对象，非静态成员函数与对象的关系就是this指针（this详细解释）。成员函数为该类所有对象共享，不仅是处于简化语言实现、节省存储的目的，而且是为了使同类对象有一致的行为。同类对象的行为虽然一致，但是操作不同的数据成员。

  9. 静态成员函数与非静态成员函数的区别

    • 无论静态函数还是非静态函数，都属于类的（这一点与数据成员的静态非静态不同），对象并不拥有函数的拷贝。
    • 调用方式：静态成员函数可以通过类或实体调用，但非静态成员函数必须用实例或实例的指针访问。
    • this: 非静态成员默认传入 this 指针，而非静态成员函数由于不属于对象而没 this 指针。
    • 作用对象：静态成员函数用于管理类相关的静态数据成员，而非静态成员函数管理具体对象相关的数据成员。
    • 多态：静态成员函数不能写成虚函数，而非静态成员函数可以。

  10. 单继承

    • 虚函数表派生类覆盖的虚函数的地址被放在了基类虚函数表对应的函数原来的位置。（覆盖）

    • 派生类没有覆盖的虚函数就延用基类的。同时，虚函数按照其声明顺序放于表中，父类的虚函数在子类的虚函数前面。

  11. 多继承

    • 每个基类都有自己的虚函数表

    • 派生类的虚函数地址存依照声明顺序放在第一个基类的虚表最后.

  12. override

    • 在子类书写的虚函数中添加，目的明确告诉编译器这个函数就是父类虚函数的一个覆盖。
    • 作用：防止手误写错了覆盖函数的签名从而引入新的虚函数。

  13. final

    • 不希望某个类被继承，或不希望某个虚函数被重写，则可以在类名和虚函数后加上 final 关键字，加上 final 关键字后，再被继承或重写，编译器就会报错。

  14. 菱形继承

    • 在继承树上，某一个子类由于多重继承，导致在内存空间中同时存在多份基类，也就是间接继承了 >= 两次的基类，菱形一般指两次。造成问题就是子类空间含有两份基类数据，当使用不同父类指针访问基类元素时候歧义。
    • 解决方法：虚继承

  15. 虚继承：

    • gcc 和 vc 实现不太一样，gcc 内存布局就是再添加一个基类指针，指向基类内存空间
    • 仍按照继承顺序排列，基类放在最后，每个类拥有自己的虚函数表。最终结果是在子类中没有重复类。因此消除了歧义，缺点是添加了基类指针，造成效率降低。

  16. static 关键字

    • 会影响变量或者函数的，作用域，生命周期，存储位置，至于怎么影响要分情况
    • 全局变量：影响了作用域，使其局限于文件本身。不影响生命周期，存储位置
    • 局部变量：改变了存储位置，生命周期，作用域不变
    • 普通函数：static使得函数在内存中只有一份，并将函数作用域限制在当前源文件。
    • 类 static 成员：static使得成员从“属于类对象”变成了“属于类，而不属于类对象”。

  17. C++ 智能指针

    • auto_ptr,unique_ptr,shared_ptr,weak_ptr。
    • 智能指针就是这样的一种类。它们的行为类似于指针，同样支持解引用* 或取成员->运算。智能指针将基本内置类型指针封装为类对象指针，管理着该指针所指向的动态分配资源，并通过类的析构函数对资源进行释放。在C++中，智能指针都是模板类，因为它们要管理的可能是用户自定义类型所分配的内存空间。

  18. malloc 和 free

    • 并不是直接向操作系统申请或者释放内存，而是应用级内存管理。
    • 最终的申请由 brk 和 mmap 进行

  19. explicit 用于限定函数不能接受隐式类型转换的函数，比如只接受 string 对象，不接受字符串

  20. 强制转换操作符

    • static_cast

      • 基本数据类型之间的转换
      • 派生类与基类类型指针的转换：注意从下向上的转换是安全的，从上向下的转换不一定安全。

    • const_cast

      • 作用是转换表达式的const性质。添加或删除
      • 用处一：改变const 指针指向的数据
      • 用处二：让 const 变量传参给非 const函数

  21. 深拷贝和浅拷贝

- 主要区别在于指针和引用的处理，是不是要深入一层拷贝

数字三角形 acwing 898

  • 思路：
    • 状态表示（集合，属性）：到达任意坐标（i, j) 的路径和的最大值为 f[i][j]
    • 状态转移 ： 这个最大值要么从左边转移过来，要么从右边转移过来，故 f[i][j] = max(f[i - 1][j - 1] , f[i - 1][j]) + a[i][j]
  • 其他注意事项：
    • 要想让三角形边界选到三角形内的数据，就要把 f 数组中三角形边界设置为负无穷
    • 题目要求是到最底层，结果要在 f 数组的最后一行中找，极端情况下，所有元素都是最小值，累积出来的路径和会很小，因此，用于遍历最后一行的值也要设无穷小。
    • 左右的坐标相对位置要找对
  • 其他解法：倒序遍历，相当于从底部走到顶部https://www.acwing.com/solution/content/128359/

最长上升子序列 acwing 895

  • 思路1
    • 状态表示（集合，属性）：以 i 结尾的最长单调子序列长度为 f[i]
    • 状态转移：遍历前面 i - 1个 f 数组元素，遇到 w[i] > w[j] 则，f[i] = max(f[i], f[j] + 1), 没遇到则 f[i] = 1
    • 时间复杂度：状态数(n) * 转移数(n)，O(n^2),
  • 思路2
    • 维护一个初始为空的递增序列 dp，遍历存储数组中的所有元素，找到其在递增序列中第一个大于等于其值的位置
      • 如果找到末尾，就在dp 末尾增加当前元素
      • 如果是中间，就替换当前位置元素
    • 复杂度分析：O(nlogn) ，状态数(n) * 转移数(logn)
    • 利用二分的解法： https://www.acwing.com/solution/content/4807/

最长公共子序列 acwing 897

  • 思路
    • 状态表示：f[i][j] A 的前 i 个字符， B 的前 j 个字符的公共子序列 的最长公共子序列
    • 状态转移：f[i][j] 可以由以下四个状态变换过来：
      • a[i],b[j] 均存在于 最长公共子序列中 (前提a[i]==b[j])
      • a[i] 在，b[j] 不在 （无前提），不完全等同于 f[i][j - 1]
      • a[i],b[j] 均不在 （无前提）
      • a[i]不在，b[j]在 （无前提）
    • 更详细的分析：https://www.acwing.com/solution/content/48820/
