    3. C++ 多态

      • 静态多态：函数重载，运算符重载，泛型多态(类模板和函数模版)。特点：编译时
      • 动态多态：也称运行时多态，主要是通过虚函数实现，允许在子类中重写父类的方法，对函数调用的解析在运行时进行

    4. 动态多态的实现机制

      • 构建时：父类将需要具体实现的方法定义为虚函数，实现共同逻辑或不实现任何逻辑（纯虚函数）。子类根据具体行为对同名虚函数进行覆盖。

      • 编译时：为处理虚函数，编译器在编译时候在子类内存空间的首地址放入虚函数表指针。虚函数表指针指向子类的虚函数表，这个虚函数表继承自父类，然后在这个虚函数表中作两个事，覆盖父类的同名虚函数地址，在末尾增加自己新定义的虚函数。

      • 运行时：用父类指针指向子类对象，想当于把子类对象当做父类对象去用，此时按照访问父类虚函数的数据流去访问，由于虚函数已经被覆盖，自然访问到的就是子函数的虚函数了。可以验证的是如果用这种方式访问子类新定义的虚函数将会出错。

        所以本质上子函数覆盖了的虚函数其实相当于生成了不同名函数，只不过编译器通过虚表把执行流引到了对应的虚函数位置。这样程序员在编码的时候就可以减少工作量了。

    5. 虚函数

      1. 虚函数是类中的成员函数，被声明为virtual。它允许派生类（子类）重写（override）基类中的虚函数，实现不同的功能。虚函数机制使得基类指针或引用可以指向派生类对象，并调用派生类中重写的函数。
      2. 类别
        1. non-virtual:
        2. virtual：可以选择性覆盖
        3. pure virture：一定要覆盖
      3. 理解：
        1. 虚函数机制其实是配合多态机制实现的
        2. 我们假设父类的函数有实函数，虚函数，实函数不会产生覆盖现象，虚函数会覆盖，且每一个实例拥有自己的虚函数表。
        3. 父类指针指向子类对象：
          1. 实函数会走父类的对应函数，调用是通过静态绑定, 直接 call 具体地址
          2. 虚函数会走子函数的对应函数，体现多态，调用是通过动态绑定
        4. 怎么实现多态的：
          1. 在每一个对象起始处放一个虚表指针，通过虚表指针找到虚表，进而找到对应的表项。
          2. 多态，虚函数，虚表都是为了实现一个目的，用一个父类指针调用成员函数，子类根据各自不同类型调用不同的虚函数实现

    6. 抽象类

      • 含有至少一个纯虚函数的类，不能被直接实例化，通常用于定义一个接口或基类。
      • 纯虚函数是一种没有实现的虚函数，它的存在是为了强制派生类提供这些函数的具体实现。

    7. 泛型编程

      • 主要通过模版机制实现，允许编写和类型无关的代码，模版可以是函数模版或类模版

    8. 函数重载

      • 参数的个数，类型，或顺序不同
      • 返回值：参数完全相同，只有返回值类型不同不能构成重载。
      • 类型：构造函数重载，运算符重载，函数重载，成员函数重载
      • 运算符重载：+ - * / == != << >> () ++ – ，C++操作符重载是允许开发者为自定义类型提供内置操作符（如+、-、*、/ ++ == +=等）的新行为的机制，通过这种方式可以使得自定义类型与内置类型一样直观和易于使用。

    9. C++ 类中的四种成员

      • 非静态数据成员被放在每一个对象体内作为对象专有的数据成员。
      • 静态数据成员被提取出来放在程序的静态数据区内(和全局变量一样的位置)，为该类所有对象共享，只存在一份。
      • 静态和非静态成员函数最终都被提取出来放在程序代码段中并为该类所有对象共享，因此每个成员函数也只能存在一份代码实体。C++中类的成员函数都是保存在静态存储区中，静态成员函数也是保存在静态存储区中，他们都是在类中保存同一个备份。
      • 说明：构成对象本身的只有数据，任何成员函数都不隶属于任何一个对象，非静态成员函数与对象的关系就是this指针（this详细解释）。成员函数为该类所有对象共享，不仅是处于简化语言实现、节省存储的目的，而且是为了使同类对象有一致的行为。同类对象的行为虽然一致，但是操作不同的数据成员。

    10. 静态成员函数与非静态成员函数的区别

      • 无论静态函数还是非静态函数，都属于类的（这一点与数据成员的静态非静态不同），对象并不拥有函数的拷贝。
      • 调用方式：静态成员函数可以通过类或实体调用，但非静态成员函数必须用实例或实例的指针访问。
      • this: 非静态成员默认传入 this 指针，而非静态成员函数由于不属于对象而没 this 指针。
      • 作用对象：静态成员函数用于管理类相关的静态数据成员，而非静态成员函数管理具体对象相关的数据成员。
      • 多态：静态成员函数不能写成虚函数，而非静态成员函数可以。

    11. 单继承

      • 虚函数表派生类覆盖的虚函数的地址被放在了基类虚函数表对应的函数原来的位置。（覆盖）

      • 派生类没有覆盖的虚函数就延用基类的。同时，虚函数按照其声明顺序放于表中，父类的虚函数在子类的虚函数前面。

    12. 多继承

    • 每个基类都有自己的虚函数表

    • 派生类的虚函数地址存依照声明顺序放在第一个基类的虚表最后.

    1. override

      • 在子类书写的虚函数中添加，目的明确告诉编译器这个函数就是父类虚函数的一个覆盖。
      • 作用：防止手误写错了覆盖函数的签名从而引入新的虚函数。

    2. final

      • 不希望某个类被继承，或不希望某个虚函数被重写，则可以在类名和虚函数后加上 final 关键字，加上 final 关键字后，再被继承或重写，编译器就会报错。

    3. 菱形继承

      • 在继承树上，某一个子类由于多重继承，导致在内存空间中同时存在多份基类，也就是间接继承了 >= 两次的基类，菱形一般指两次。造成问题就是子类空间含有两份基类数据，当使用不同父类指针访问基类元素时候歧义。
      • 解决方法：虚继承

    4. 虚继承：

      • gcc 和 vc 实现不太一样，gcc 内存布局就是再添加一个基类指针，指向基类内存空间
      • 仍按照继承顺序排列，基类放在最后，每个类拥有自己的虚函数表。最终结果是在子类中没有重复类。因此消除了歧义，缺点是添加了基类指针，造成效率降低。

    5. static 关键字

      • 会影响变量或者函数的，作用域，生命周期，存储位置，至于怎么影响要分情况
      • 全局变量：影响了作用域，使其局限于文件本身。不影响生命周期，存储位置
      • 局部变量：改变了存储位置，生命周期，作用域不变
      • 普通函数：static使得函数在内存中只有一份，并将函数作用域限制在当前源文件。
      • 类 static 成员：static使得成员从“属于类对象”变成了“属于类，而不属于类对象”。

    6. static 关键字：static关键字用于控制变量或函数的存储期和作用域

      1. 局部静态变量：在函数内部声明为static的变量，其生命周期贯穿整个程序，只在首次调用函数时初始化一次。
      2. 全局静态变量：在函数外部声明为static的变量，其作用域限定在定义它的文件内，不会被其他文件访问。
      3. 静态成员变量：类的static成员变量属于类本身，而不是类的某个特定对象，所有对象共享同一份数据。
      4. 静态成员函数：类的static成员函数不依赖于类的任何特定对象，只能访问类的static成员和其他static成员函数。所有对象共享同一份源码。调静态成员函数可以用对象实例也可以用类名
      5. 静态类：声明为static的类只能在定义它的文件内使用，不能被其他文件访问。

    9. C++ 智能指针

      • auto_ptr,unique_ptr,shared_ptr,weak_ptr。
      • 智能指针就是这样的一种类。它们的行为类似于指针，同样支持解引用* 或取成员->运算。智能指针将基本内置类型指针封装为类对象指针，管理着该指针所指向的动态分配资源，并通过类的析构函数对资源进行释放。在C++中，智能指针都是模板类，因为它们要管理的可能是用户自定义类型所分配的内存空间。

    11. new/delete

      1. new 先分配在构造
      2. delete 先析构，再释放
      3. array new 要记得用 array delete, array new 分配内存的时候开始部分会有四个字节记录数组长度

    12. new和malloc的区别：

      1. 构造函数和析构函数：new在分配内存的同时调用对象的构造函数，而malloc仅分配内存，不调用构造函数。相应地，delete会调用析构函数，而free不会。
      2. 类型安全：new是类型安全的，返回一个具体类型的指针，而malloc返回void*类型，需要显式类型转换。
      3. 表达式：new是一个运算符，可以与类型一起使用，如new int；malloc是一个函数，需要指定分配的字节大小，如malloc(sizeof(int))。
      4. 错误处理：new在内存分配失败时抛出std::bad_alloc异常（除非使用了new (nothrow)），而malloc在失败时返回NULL。
      5. 重载：new和delete运算符可以被重载，允许自定义内存分配和释放的行为；malloc和free则不能被重载，它们是C标准库函数。
      6. 使用场景：new和delete通常用于C++对象的动态分配和释放，而malloc和free更适用于分配基本数据类型的数组或需要手动管理内存大小的场景。

    13. new 出来的 array, 也即动态内存中的数组，一定要用 delete [] array 进行删除，否则只会释放第一个元素

        • 基本数据类型之间的转换
        • 派生类与基类类型指针的转换：注意从下向上的转换是安全的，从上向下的转换不一定安全。

      • const_cast

        • 作用是转换表达式的const性质。添加或删除
        • 用处一：改变const 指针指向的数据

    20. 类模版，函数模版：

      1. 允许定义类和函数的时候不指定数据类型，而是用类型参数。这样同一个类模板可以用于多种数据类型，而不需要为每种类型重写类定义。类模板通过关键字template和尖括号<>来定义。类模板和函数模板的实例化是在编译时进行的，编译器根据传入的参数类型生成具体的类或函数实现。
      2. 设计类和模版的时候，你认为那些类型可以抽出来允许用户任意指定，那就将其作为模版参数允许用户指定

    21. 内存泄漏：程序通过 new 申请的内存空间，却没有通过 delete 释放，空间依然保留，原来指向它的指针却失效，造成无法使用。

    27. 临时对象：指在表达式中创建并使用后立即销毁的对象，它们通常用于函数返回值、参数传递、以及在表达式中生成的临时值。临时对象的生命周期只在包含它们的完整表达式内有效，且不能被直接引用或赋值给变量。语法：typename()

    28. inline：C++中的inline关键字是一种函数修饰符，用于建议编译器在编译时将函数体直接嵌入到每个调用点，以减少函数调用的开销，提高程序的执行效率。它通常用于实现小型、频繁调用的函数，如访问器和设置器。inline函数的内联展开由编译器决定，编译器可能会忽略内联请求。

    29. 智能指针：C++智能指针是一种自动管理内存分配和释放的模板类，它通过重载指针操作符来模拟指针的行为，同时在适当的时候自动释放分 配的内存，以防止内存泄漏。常见的智能指针类型包括 std::unique_ptr、std::shared_ptr 和 std::weak_ptr。

    39. Lambda 函数

      • 介绍：匿名函数，可以作为一个参数传入接收匿名函数或仿函数的位置，就地写一个小的函数，方便且优雅。lambda 实际执行中被转变成函数对象 。

      • 基本语法：[capture](parameter)-> return_type { /*....*/ }

      • [capture] : [] 内为外部变量的传递方式。

      • (parameters) ：（）内为形参，和普通函数的形参一样。

      • -> return_type：返回类型，编译器自动推断，可省略。

      • 参考文献：C++中的Lambda函数详解

      • 基本语法：struct foo { int operator()(args) const { /*...*/ }};
